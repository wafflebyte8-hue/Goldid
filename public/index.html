Install

Run it from PowerShell, Bash, or the desktop launcher.

Windows

irm https://raw.githubusercontent.com/wafflebyte8-hue/Goldid/main/setup.ps1 | iex

Installs to `C:\goldid`, registers `gd`, installs Electron, and creates Desktop + Start Menu shortcuts.

Linux and macOS

curl -fsSL https://raw.githubusercontent.com/wafflebyte8-hue/Goldid/main/setup.sh | bash

Installs the CLI on Linux and macOS. Linux also gets the desktop app and app-menu entry. Remove it later with `~/.local/share/goldid/uninstall.sh`.

Update

gd update check
gd update
gd update --force

Checks GitHub with cache-resistant version lookup and reruns the installer without touching `~/.goldid` data.

Skills

Download reusable skill packs from the community.

GolDid skills are portable folders with `SKILL.md` and `Version.js`. Each published skill gets a unique 4-digit id. Browse `/skills/`, open a detail page at `/skills/<id>/`, then install by id from the CLI or desktop.

Desktop

A focused Electron shell for the same GolDid core.

The desktop app shares config, memory, sessions, skills, providers, sandbox settings, image setup, and update checks with the CLI. It adds Markdown rendering, a command menu, settings dialogs, sidebar browsing, and inline tool approvals. Linux launchers now force GTK 3 and write startup failures to `~/.goldid/desktop.log`.

Security

Practical guardrails for local agent work.

Keys at rest: API keys are encrypted with AES-256-GCM, with TPM sealing where available and machine-bound fallback elsewhere.

Dangerous tools: shell, write_file, and generate_image can require explicit approval, with exact arguments shown before execution.

Sandboxing: jail mode confines file and shell tools to the launch directory; docker mode gives stronger shell isolation.

Desktop hardening: context isolation, no node integration, a narrow preload API, DOMPurify for assistant Markdown, and escaped user/file content.
